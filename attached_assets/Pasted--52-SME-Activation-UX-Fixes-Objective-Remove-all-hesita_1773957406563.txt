#52 — SME Activation UX Fixes

## Objective

Remove all hesitation, ambiguity, and cognitive load so a first-time SME user can:

- Sign up
- Complete onboarding
- Add their first ESG data point
- Understand what they’ve done
- Generate a first report

All in one uninterrupted 10–15 minute session without assistance.

This is a UX, copy, and usability optimisation. No new features. Improve clarity and flow only.

Core rule:
Every screen must clearly answer:
1. What is this?
2. Why does it matter?
3. What should I do next?

This task supersedes draft tasks #5, #6, #7, #19, and #24. Do not implement those separately.

---

## Required approach

1. Audit first (mandatory)
   - Review all listed screens
   - Identify:
     - unclear wording
     - missing guidance
     - dead ends
     - hidden validation issues

2. Implement changes in the order below

3. Use the exact copy provided
   - Do not reword unless technically required

4. After implementation:
   - Run full journey with a fresh account
   - Fix any hesitation points before marking complete

Do not skip steps. Do not partially implement.

---

## Done looks like

A new SME user must:

- Not hesitate at any step
- Not be confused by wording
- Always know what to do next
- Complete onboarding smoothly
- Add first data point without uncertainty
- Reach dashboard with a clear next action

If the user pauses or asks “what do I do next?” — the task is not complete.

---

## Out of scope

- New features or new data models
- New reporting frameworks or metrics
- Admin-side changes
- Changes to calculation or scoring logic

## Global requirements (apply everywhere)

- All errors must:
  - explain what went wrong
  - explain how to fix it

- No silent failures
  - every action shows success OR error

- All async actions:
  - must show loading state

- All CTAs:
  - must route correctly
  - no dead links

---

## Final validation (required)

Test full journey with a fresh user:

- Sign up
- Complete onboarding
- Add one data point
- Reach dashboard

The user must:
- never hesitate
- never be confused
- always know the next step

If the user stops or asks “what do I do next?” → task is incomplete.
