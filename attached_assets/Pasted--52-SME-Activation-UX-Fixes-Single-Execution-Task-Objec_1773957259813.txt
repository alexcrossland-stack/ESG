#52 — SME Activation UX Fixes (Single Execution Task)

## Objective

Remove all hesitation, ambiguity, and cognitive load so a first-time SME user can:

- Sign up
- Complete onboarding
- Add their first ESG data point
- Understand what they’ve done
- Generate a first report

All in one uninterrupted 10–15 minute session without assistance.

This is a UX, copy, and usability optimisation. Do not build new features. Improve clarity and flow only.

Core rule:
Every screen must clearly answer:
1. What is this?
2. Why does it matter?
3. What should I do next?

---

## Required approach

1. Audit all listed screens first
2. Then implement changes in the order below
3. Use the exact copy provided
4. Validate full journey at the end with a fresh user

Do not skip steps. Do not generalise copy.

---

## Implementation

---

### 1. Sign-up improvements

- Default tab = Create Account
- Add secondary link: “Already have an account? Sign in”

Update fields:

Full name  
Placeholder: e.g. Jane Smith

Password helper:
At least 6 characters

Add above legal checkboxes:
By creating an account, you agree to our terms and privacy policy

Loading state:
Setting up your account…

After submit (NEW transition screen):
Account created — setting up your workspace…

### 2. Onboarding entry (path choice)

Let’s set up your ESG tracking

We’ll guide you through everything you need to get started

Set it up for me

### 3. Company profile step

Rename:
Operational Profile → What type of business are you?

- Scroll to it

### 4. ESG maturity step

### 5. ESG priorities (reduce overload)

Most businesses like yours start with these

### 9. Action plan & completion

### 10. Dashboard first-run

Replace empty scores:

---

## Global requirements

- All errors must explain what went wrong and how to fix it
- No silent failures — every action shows success or error
- All async actions show loading state
- All CTAs must route correctly (no dead links)

---

## Final validation (required)

Test with a new user:

User must:
- not hesitate at any step
- not be confused by wording
- always know what to do next
- complete onboarding smoothly
- add first data point without uncertainty
- reach dashboard with a clear next action

If the user pauses or asks “what do I do next?” → the task is not complete.
