Build a production-ready onboarding system for the ESG app for SME users.

Goal

Create a first-time user onboarding flow that lets users choose between:
	1.	Guided setup
	2.	Manual setup

This must be fully functional, with frontend, backend, database persistence, and integration into the existing ESG modules.

⸻

Core behaviour

First login / first company setup

When a new company account is created or a user logs in for the first time, show an onboarding entry screen.

Display:
	•	Title: Welcome to your ESG workspace
	•	Subtitle explaining the app helps SMEs set up ESG metrics, policies, carbon tracking, supplier assessments, and reporting
	•	Primary button: Guided Setup
	•	Secondary button: Set Up Manually

Do not send first-time users straight to the dashboard without this choice.

⸻

Path 1: Guided setup

Build a multi-step onboarding wizard with:
	•	progress bar
	•	previous / next navigation
	•	autosave on every step
	•	ability to exit and resume later
	•	mobile responsive layout
	•	plain English helper text

Guided setup steps

Step 1: Company profile
Collect and save:
	•	company name
	•	industry
	•	business type
	•	employee count
	•	number of sites
	•	country
	•	reporting year start
	•	operational profile: office / manufacturing / logistics / retail / mixed

Use this data to drive recommendations later.

Step 2: ESG maturity
Collect one value:
	•	just starting
	•	some policies in place
	•	formal ESG programme

Use this to adjust defaults:
	•	just starting = simpler setup and lighter defaults
	•	some policies = standard defaults
	•	formal programme = advanced setup and more metrics

Step 3: Module selection
Let the user enable or disable:
	•	ESG Metrics Library
	•	Carbon Calculator
	•	Policy Generator
	•	Supplier Questionnaire
	•	Reporting Dashboard
	•	Traffic Light Scoring

Default core modules to enabled.

Step 4: Metric selection
Show recommended default SME metrics grouped by category.

Environmental:
	•	Electricity Consumption
	•	Gas / Fuel Consumption
	•	Scope 1 Emissions
	•	Scope 2 Emissions
	•	Waste Generated
	•	Recycling Rate
	•	Water Consumption

Social:
	•	Employee Headcount
	•	Gender Diversity
	•	Employee Turnover
	•	Training Hours
	•	Health & Safety Incidents

Governance:
	•	Board Diversity
	•	ESG Policy Adoption
	•	Supplier ESG Screening

Requirements:
	•	preselect recommended metrics
	•	allow deselect
	•	allow future expansion later in settings
	•	include short plain-English descriptions
	•	recommendations should vary based on company profile and maturity

Step 5: Carbon calculator setup
Ask:
	•	do you track electricity use?
	•	do you track gas or fuel use?
	•	do you have company vehicles?
	•	do you want automatic Scope 1 and Scope 2 emissions calculations?

If enabled:
	•	link electricity to Scope 2 calculations
	•	link fuel and vehicle usage to Scope 1 calculations
	•	mark Scope 1 and Scope 2 as derived metrics
	•	automatically recalculate derived values when source inputs change

Design this so emission factors are stored centrally and can be updated later by admin.

Step 6: Policy generation setup
Ask whether the user wants starter ESG policy drafts created.

If yes, generate editable drafts for:
	•	Environmental Policy
	•	Sustainability Policy
	•	Health & Safety Policy
	•	Diversity & Inclusion Policy
	•	Supplier Code of Conduct
	•	Anti-Bribery / Ethics Policy
	•	ESG Commitment Statement

These should be created using saved company profile data.

Step 7: Supplier assessment setup
Ask whether supplier assessment is needed.

If yes:
	•	enable supplier module
	•	create a default supplier ESG questionnaire
	•	enable supplier scoring
	•	prepare supplier workflow for later invitations and responses

Step 8: Completion
Show a completion summary with:
	•	number of metrics enabled
	•	whether carbon calculator is active
	•	number of policy drafts created
	•	whether supplier assessment is enabled
	•	whether traffic light scoring is enabled

Primary CTA:
	•	Go to Dashboard

⸻

Path 2: Manual setup

If the user selects manual setup:
	•	do not force the wizard
	•	take them to the dashboard
	•	show a setup checklist card at the top
	•	track checklist completion

Checklist items:
	•	Add company profile
	•	Enable ESG metrics
	•	Configure carbon calculator
	•	Generate policies
	•	Set up supplier questionnaire
	•	Review traffic light scoring

Each item should link to the correct configuration page.

⸻

Required database changes

Add onboarding state to the company or workspace record.

Fields required:
	•	onboarding_complete boolean
	•	onboarding_path enum(‘guided’,‘manual’)
	•	onboarding_step integer
	•	onboarding_progress_percent integer
	•	onboarding_started_at datetime
	•	onboarding_completed_at datetime

Add onboarding selections storage:
	•	selected_modules json
	•	selected_metrics json
	•	esg_maturity string
	•	onboarding_answers json

If not already present, create or extend related tables for:
	•	company_profile
	•	company_metric_settings
	•	company_module_settings
	•	company_policy_documents
	•	supplier_questionnaire_templates
	•	emission_factors
	•	traffic_light_thresholds

⸻

Required business logic

Recommendation engine

Build simple rules to recommend defaults from company profile + maturity.

Examples:
	•	manufacturing: prioritise waste, fuel, water, safety
	•	office-based: prioritise electricity, travel, training, diversity, policies
	•	logistics: prioritise fuel, vehicles, Scope 1, safety
	•	smaller companies: show lighter defaults
	•	advanced maturity: enable more governance and reporting options

Derived metric engine

Support metrics with:
	•	raw/manual input
	•	calculated/derived values

Examples:
	•	Scope 1 = fuel + vehicle emissions logic
	•	Scope 2 = electricity x factor
	•	recycling rate = recycled waste / total waste x 100

Derived metrics must:
	•	recalculate automatically
	•	be tagged as derived
	•	be read-only unless explicitly overridden by admin logic

Traffic light scoring

Create default threshold rules with later admin edit capability.

Support:
	•	green
	•	amber
	•	red

Scoring should work for:
	•	raw metrics
	•	derived metrics
	•	supplier scores where relevant

Backend/API requirements

Create backend endpoints or handlers for:
	•	get onboarding state
	•	start onboarding
	•	save onboarding step
	•	resume onboarding
	•	complete onboarding
	•	get recommendations
	•	create default policies
	•	create default questionnaire

Ensure idempotency where possible so users can resume without duplicate data creation.

⸻

Dashboard integration

After onboarding, the dashboard must reflect the selected setup.

Examples:
	•	carbon calculator enabled → show carbon widgets and derived emissions metrics
	•	policies enabled → show draft policy section
	•	supplier module enabled → show supplier section
	•	incomplete setup → show continue setup prompt
	•	manual path → show checklist until setup is complete
