The goal is to create a simple but professional ESG tracking system for small and medium-sized businesses. The feature must support manual metrics, automatic calculations, derived metrics, trend tracking, and traffic light scoring.

The solution must be clean, easy to use, and future-ready for reports, benchmarking, and AI recommendations.

⸻

1. Core outcome required

Create an ESG metrics system that allows a business to:
	•	track Environmental, Social, and Governance metrics
	•	enter raw source data monthly, quarterly, or annually
	•	automatically calculate selected ESG metrics from raw data
	•	generate derived metrics from other saved values
	•	compare current value versus previous period and target
	•	assign a red / amber / green traffic light status to each metric
	•	show trend history and dashboard summaries
	•	support admin configuration of metrics, targets, and settings

The system must be designed so that it stores both raw data and calculated outputs, not just final KPI values.

5. Automatic calculation rules

Implement the following formulas.

Scope 1 Emissions

Calculate using gas/fuel consumption and company vehicle fuel use.

Formula:
	•	gas or fuel consumption × relevant emission factor
	•	company vehicle fuel litres × relevant emission factor
	•	sum both values

Scope 2 Emissions

Formula:
	•	electricity consumption in kWh × electricity emission factor

Recycling Rate

Formula:
	•	recycled waste / total waste × 100

Business Travel Emissions

Calculate from travel records using travel-type emission factors.
Support:
	•	flights
	•	rail
	•	hotel stays
	•	car mileage
	•	taxis

Employee Turnover Rate

Formula:
	•	employee leavers / average headcount × 100

Absence Rate

Formula:
	•	absence days / total working days × 100

Data Privacy Training Completion

Formula:
	•	trained staff / total staff × 100

Supplier Code of Conduct Adoption

Formula:
	•	suppliers signed code / total suppliers × 100

Living Wage Coverage

Formula:
	•	employees paid living wage / total employees × 100

Training Hours per Employee

Formula:
	•	total training hours / total employees

7. Traffic light scoring logic

Add a red / amber / green status for every metric.

Each metric must support one of these direction settings:
	•	higher_is_better
	•	lower_is_better
	•	target_range
	•	compliance_yes_no

Green

Set status to green when:
	•	metric meets or beats target
	•	or trend improved in the correct direction

Amber

Set status to amber when:
	•	metric is close to target, such as within 5 percent
	•	or no significant movement versus prior period

Red

Set status to red when:
	•	metric misses target materially
	•	or trend moves in the wrong direction

Compliance metrics

For yes/no metrics:
	•	yes = green
	•	no = red

Target range metrics

For metrics such as board meetings held:
	•	green if within range or above minimum threshold
	•	amber if slightly below range
	•	red if materially below range

Make thresholds configurable in admin settings.

11. Business logic requirements

Data storage

Do not only store final ESG metric values.

Store:
	•	raw source data
	•	formulas
	•	emission factors
	•	calculated outputs
	•	historical targets
	•	traffic light history
	•	notes
	•	audit trail

Recalculation

When raw data changes, dependent calculated metrics must automatically update.

Example:
If electricity consumption changes, Scope 2 Emissions and Carbon Intensity should also update.

Validation

Add validation for:
	•	missing required source data
	•	divide-by-zero cases
	•	invalid percentages above 100 where not allowed
	•	invalid negative values unless explicitly supported
