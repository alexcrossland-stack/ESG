ESG Metrics Expansion + Framework Mapping Implementation Scope

Build the ESG platform so it remains ESG-focused, uses the existing 28 core metrics as the default baseline, and expands into a stronger framework-aligned ESG system without becoming overly complex for SMEs.

The objective is to make the platform:
	•	strong for core ESG measurement
	•	expandable for GRI / ISSB / TCFD / ESRS-style alignment
	•	still practical for SMEs
	•	structured so metrics, evidence, policies, targets, and framework mapping all work together

⸻

1. Core product approach

Keep the existing 28 metrics as the universal default ESG baseline.

Add a second layer of advanced metrics and a third layer of framework mapping + non-metric ESG controls.

The platform should therefore have 4 layers:

Layer 1 — Core metrics

The existing 28 standard ESG metrics remain enabled by default for all businesses.

Layer 2 — Expanded metrics

Add a controlled set of additional ESG metrics to improve framework coverage.

Layer 3 — Framework mapping

Allow each metric to be mapped to one or more frameworks, standards, or disclosure areas.

Layer 4 — ESG management controls

Add non-metric ESG capability: materiality, targets, action plans, policies, evidence, and risk registers.

	•	company-level
	•	site-level
	•	optional department/business-unit level later

Ensure all relevant metric tables support:
	•	business_id
	•	site_id nullable

Rollups required:
	•	site value
	•	company total
	•	company weighted average where relevant
	•	group comparisons between sites

11. Scoring model

Implement a more mature scoring system that separates:
	•	data completeness
	•	performance
	•	governance maturity
	•	framework readiness

Do not use a single opaque ESG score only.

Create score outputs such as:

A. Core ESG completeness score

Measures how much required data is present.

B. ESG performance score

Based on actual metric values and trends.

C. Management maturity score

Based on policies, ownership, targets, actions, evidence, and review cycles.

D. Framework readiness score

Per framework:
	•	GRI readiness
	•	ISSB readiness
	•	TCFD readiness
	•	ESRS readiness

Each readiness score should show:
	•	covered items
	•	partial items
	•	missing items

13. AI / assistant behaviour

Update the assistant logic so it no longer treats ESG only as metric collection.

The assistant should be able to:
	•	explain what each metric means
	•	suggest missing metrics based on sector and frameworks selected
	•	identify gaps in evidence
	•	recommend policies or targets to improve readiness
	•	summarise framework coverage without claiming formal compliance
	•	generate draft ESG action plans
	•	surface overdue reviews, missing owners, missing risk controls

17. Key product rule

A very important implementation rule:

Do not build this as a “full compliance engine.”
Build it as an:
	•	ESG measurement platform
	•	ESG management system
	•	framework-readiness and alignment platform

That keeps the product credible and commercially strong.

⸻

18. Acceptance criteria

Done looks like:
	•	the original 28 metrics remain intact as the default core set
	•	at least 25–30 advanced metrics can be enabled and collected
	•	metrics are stored via reusable definitions, not hardcoded only
	•	derived metrics recalculate automatically
	•	each metric can map to one or more frameworks
	•	businesses can select frameworks they care about
	•	framework dashboards show covered / partial / missing
	•	materiality assessment influences recommendations
	•	policies, targets, actions, and risk registers are first-class features
	•	evidence can be attached to metric submissions
	•	site-level data can roll up to company-level views
	•	scoring is split into completeness, performance, maturity, and readiness
	•	reports/export reflect both metrics and ESG management controls

⸻

19. Final instruction to Replit

Implement this as a modular ESG architecture where metrics are only one layer of the product. Preserve the SME-friendly experience, but make the backend and data model robust enough to support framework alignment, evidence, governance, targets, action plans, risk tracking, and future sector-specific expansion.
