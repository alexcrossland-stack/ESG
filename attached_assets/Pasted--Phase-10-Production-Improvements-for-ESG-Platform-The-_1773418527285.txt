# Phase 10 — Production Improvements for ESG Platform

The ESG platform is now deployed and working on the production server.  
Implement the following improvements to increase reliability, usability, collaboration, and commercial readiness.

Follow existing project architecture (React + Express + PostgreSQL).  
Maintain TypeScript types and do not break existing APIs.

--------------------------------
1. Error Monitoring (Sentry)
--------------------------------

Goal: Capture backend and frontend errors automatically.

Install dependencies:

npm install @sentry/node @sentry/tracing
npm install @sentry/react

Backend setup:

Update server/index.ts

Initialize Sentry before Express routes:

import * as Sentry from "@sentry/node";

Sentry.init({
  dsn: process.env.SENTRY_DSN,
  tracesSampleRate: 0.2
});

Add middleware:

app.use(Sentry.Handlers.requestHandler());
app.use(Sentry.Handlers.errorHandler());

Ensure all uncaught exceptions are logged.

Frontend setup:

Update client/src/main.tsx

Initialize Sentry React to capture:

- React render errors
- API failures
- unhandled promise rejections

Add environment variable:

SENTRY_DSN

2. Product Analytics (PostHog)

Install:

npm install posthog-js

client/src/main.tsx

POSTHOG_KEY

- onboarding_completed
- policy_created
- metric_value_entered

- report_generated
- assistant_question_asked
- dashboard_viewed

Example event:

posthog.capture("policy_created")

3. ESG Setup Progress Tracker
--------------------------------

Goal: Help SMEs understand how far they are through ESG setup.

Create component:

client/src/components/esg-progress.tsx

Checklist items:

- Company profile completed
- ESG priorities selected
- First policy created
- First metric data entered
- Evidence uploaded
- First report generated

Create backend endpoint:

GET /api/programme/progress

Return:

{
 profileComplete: boolean,
 prioritiesSelected: boolean,
 policiesCreated: number,
 metricsWithData: number,
 evidenceUploaded: number,
 reportsGenerated: number
}

Display progress bar in dashboard.

File:

client/src/pages/dashboard.tsx

Place above the Programme Status card.

4. Multi-User Company Accounts
--------------------------------

Goal: Allow multiple users per company workspace.

Create database table:

company_users

Fields:

id
company_id
user_id
role
created_at

Roles:

admin
editor
viewer

Update auth logic so users belong to companies through company_users.

Create endpoint:

POST /api/company/invite

Flow:

Admin enters email → invite link generated → user joins company workspace.

Create new page:

client/src/pages/team.tsx

Admins can:

- invite users
- change roles
- remove users

6. Automated Database Backups
--------------------------------

Create backup script:

script/backup-db.sh

Command:

pg_dump $DATABASE_URL > /var/backups/esg-$(date +%F).sql

Keep last 14 backups.

Add cron job:

0 2 * * * /root/ESG/script/backup-db.sh

Log backup success.
