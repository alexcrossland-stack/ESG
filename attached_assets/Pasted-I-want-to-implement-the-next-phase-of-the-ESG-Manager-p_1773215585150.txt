I want to implement the next phase of the ESG Manager platform to make it more valuable for SME customers and improve long-term product stickiness. The focus should be on workflows, outputs that customers actually need, and operational usability.

Please implement the following improvements.

1. Add task ownership and workflow views

Allow key items in the system to be assigned to a specific user.

The following items must support an owner, due date, and status:
	•	metrics
	•	raw data submissions
	•	actions
	•	policies
	•	questionnaire responses
	•	evidence requests

Add two new dashboard views:

My Tasks
	•	shows all tasks assigned to the user
	•	shows due date and overdue status
	•	includes metrics to update, evidence requests, policy reviews, and actions

My Approvals
	•	shows items waiting for approval
	•	includes policies, reports, questionnaire answers, and locked submissions

Admins must be able to reassign owners.

⸻

2. Add recurring reporting periods

Introduce structured reporting cycles.

Support:
	•	monthly
	•	quarterly
	•	annual reporting periods

Each new period should automatically:
	•	copy forward active metrics
	•	copy targets
	•	keep policy review schedules
	•	maintain action plans
	•	maintain site structure if applicable

Add the ability to compare:
	•	current period vs previous period
	•	current period vs same period last year

Show trends on the dashboard.

⸻

3. Build a Board ESG Report generator

Add a one-click Board ESG Summary Report.

The report should include:
	•	ESG score and trend
	•	category performance (Environmental, Social, Governance)
	•	carbon summary
	•	key risks and red metrics
	•	progress against targets
	•	overdue actions
	•	policy status
	•	upcoming review items
	•	key recommendations

The report should be exportable to PDF and DOCX.

⸻

4. Build a Customer ESG Response Pack

Create a feature that generates a Customer ESG Response Pack.

This should include:
	•	company ESG profile
	•	key ESG metrics
	•	carbon emissions summary
	•	list of approved ESG policies
	•	modern slavery statement
	•	anti-bribery commitment
	•	data protection commitment
	•	governance summary
	•	evidence references where available

This pack should help SMEs respond to customer procurement or ESG questionnaires.

Allow export to PDF and DOCX.

⸻

5. Create a reusable ESG questionnaire answer bank

Add a library that stores commonly used ESG answers.

Features required:
	•	store common customer ESG questions
	•	store approved standard answers
	•	link answers to policies, metrics, or evidence
	•	detect duplicate questions
	•	show when an answer needs review because underlying data changed
	•	allow answers to be reused in future questionnaires

This should integrate with the existing questionnaire autofill feature.

⸻

6. Add evidence request workflows

Improve evidence management by allowing admins to request documents.

Admins should be able to:
	•	request evidence from a specific user
	•	specify which metric, policy, or questionnaire item the evidence supports
	•	set a due date
	•	track evidence status

Statuses should include:
	•	requested
	•	uploaded
	•	under review
	•	approved
	•	rejected
	•	expired

Add an Evidence Library where approved documents can be reused across reports and questionnaires.

⸻

7. Add multi-site support

Allow companies to manage multiple locations.

Add a Sites / Locations structure where:
	•	each site can have its own data
	•	metrics can be entered per site
	•	results roll up to company totals
	•	actions and policies can be site-specific or company-wide

This will allow the platform to support larger SMEs.

⸻

8. Improve policy management

Upgrade the policy system so it behaves more like a proper document control system.

Add the following features:
	•	policy review dates
	•	approval history
	•	version comparison
	•	policy register
	•	policy applicability by site or department
	•	employee read-and-acknowledge workflow

The system should track who has acknowledged each policy.

⸻

9. Build a platform super-admin console

Create a super-admin dashboard for the platform operator.

This should allow viewing:
	•	all companies using the platform
	•	onboarding progress
	•	number of users per company
	•	module usage
	•	AI usage
	•	failed imports or background jobs

Allow super-admin users to:
	•	deactivate companies
	•	reset onboarding
	•	review audit logs
	•	support customers if needed

⸻

10. Add subscription and plan controls

Prepare the system for SaaS subscriptions.

Add plan controls so companies can have different limits based on their plan.

Examples:
	•	number of users
	•	number of policies
	•	number of reports
	•	evidence storage limits
	•	AI usage limits
	•	module availability

Add an upgrade prompt system when companies reach their limits.

⸻

Goal of these changes

These improvements should make the platform:
	•	easier for SMEs to use operationally
	•	better at producing outputs they need for customers and boards
	•	more difficult to replace once adopted
	•	scalable as a commercial SaaS product.
