Implement a Platform Super Admin System for the ESG Manager SaaS so the platform owner can manage all companies, users, and platform operations from a central admin console.

The system must integrate with the existing authentication, company isolation model, and health monitoring already present in the application.

------------------------------------------------

OBJECTIVE

Create a Super Admin Console that allows platform operators to:

• View and manage all companies  
• View and manage all users  
• Impersonate company admins for support  
• Suspend or reactivate companies  
• Monitor platform activity  
• Access system health data  

This console must be secure and restricted to users with the role super_admin.

------------------------------------------------

ROLE MODEL

Update the role system to support three levels:

super_admin  
company_admin  
user  

Rules:

super_admin
• platform-level access
• can view all companies
• can impersonate company admins
• can access platform health and billing
• cannot bypass audit logging

company_admin
• manages their company users
• manages policies, metrics, reports
• manages billing for their company

user
• standard user access
• data entry, evidence uploads, reporting

------------------------------------------------

DATABASE CHANGES

Update the users schema:

users table must include:

role enum
("super_admin", "company_admin", "user")

Default role = "user"

Ensure at least one user can be assigned super_admin manually through database seed or migration.

------------------------------------------------

SECURITY

Create middleware:

requireSuperAdmin

This middleware must verify:

req.user.role === "super_admin"

All admin routes must use this middleware.

------------------------------------------------

ADMIN ROUTES

Create new admin API endpoints in server/routes.ts.

1. List companies

GET /api/admin/companies

Return:

id  
name  
industry  
country  
planTier  
userCount  
policyCount  
metricCount  
reportCount  
createdAt  
lastActive  

2. List users across platform

GET /api/admin/users

Return:

id  
name  
email  
companyName  
role  
lastLogin  
status  

3. Company details

GET /api/admin/company/:companyId

Return full company overview including:

users  
policies  
metrics  
reports  
planTier  
maturityLevel  

4. Suspend company

POST /api/admin/company/suspend

Input:

companyId

Effect:

company.status = "suspended"

Suspended companies cannot login.

5. Reactivate company

POST /api/admin/company/reactivate

Input:

companyId

Effect:

company.status = "active"

6. Impersonate company admin

POST /api/admin/impersonate/:companyId

Creates an impersonation session allowing the super admin to view the platform as that company.
