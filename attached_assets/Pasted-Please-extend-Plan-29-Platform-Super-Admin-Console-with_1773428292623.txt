Please extend Plan #29 (Platform Super Admin Console) with the following three additional capabilities to make the admin console production-grade for operating a SaaS platform.

------------------------------------------------
30 - Tenant Diagnostics Panel
------------------------------------------------

What & Why

When supporting customers, platform operators often need to quickly diagnose why a company is struggling or experiencing errors. A tenant diagnostics panel allows super admins to instantly view system health and activity for a specific company without querying the database manually.

Done looks like

• Visiting /admin/companies/:companyId shows a "Tenant Diagnostics" section
• The diagnostics panel displays:
  - last login timestamp
  - number of active users
  - policies created
  - metrics enabled
  - latest metric entry date
  - evidence files uploaded
  - reports generated
  - AI assistant usage count
  - last error recorded for that company
• A "Recent Activity" timeline shows the last 20 events for that company (policy created, metric entered, report generated, etc.)
• A "System Health" section shows if the company has missing onboarding steps (no policies, no metrics, no evidence)

Tasks

1. Backend diagnostics endpoint

Add endpoint:

GET /api/admin/company/:companyId/diagnostics

Return:

{
  company,
  stats: {
    users,
    policies,
    metrics,
    reports,
    evidence,
    lastMetricEntry,
    lastLogin
  },
  recentActivity[],
  lastErrors[]
}

Use existing tables where possible.

2. Activity timeline

Create helper to collect activity from:
- policies
- metric_values
- evidence_files
- reports

Return the latest 20 events ordered by timestamp.

3. Admin UI

Add "Diagnostics" section to:

client/src/pages/admin-company.tsx

Display stats as cards and the activity timeline below.

31 - Platform Revenue & Subscription Dashboard
------------------------------------------------

What & Why

Operators need a high-level view of platform revenue and subscription growth without logging into Stripe. A revenue dashboard inside the admin console helps track platform performance.

Done looks like

Admin dashboard (/admin) includes a "Platform Revenue" section showing:

• total companies
• free vs pro companies
• monthly recurring revenue (MRR)
• subscriptions created last 30 days
• churned subscriptions last 30 days
• revenue chart (last 6 months)

Tasks

1. Backend endpoint

Add:

GET /api/admin/revenue

Return:

{
  totalCompanies,
  freeCompanies,
  proCompanies,
  mrr,
  subscriptionsLast30Days,
  churnLast30Days,
  monthlyRevenue[]
}

Use Stripe subscription data already stored on companies table.

2. Revenue chart

Use recharts to render:

Monthly revenue last 6 months.

File:

client/src/pages/admin.tsx

32 - Platform Health & System Monitoring
------------------------------------------------

What & Why

Super admins need visibility into the operational health of the platform. This includes error rates, API performance, and system load.

Done looks like

Admin dashboard contains a "Platform Health" section showing:

• API error rate (last 24h)
• slow routes (>2s)
• failed jobs
• AI API failures
• email send failures
• database connection health

Tasks

1. Backend endpoint

Add:

GET /api/admin/platform-health

Return:

{
  errorRate24h,
  slowRoutes[],
  failingJobs[],
  aiFailures[],
  emailFailures[],
  dbStatus
}

Use existing health_events table.

2. Admin health dashboard

Add a "Platform Health" card to the admin dashboard displaying:

• error count last 24h
• slowest routes
• recent failures

File:

client/src/pages/admin.tsx

------------------------------------------------
Additional Safety Requirements
------------------------------------------------

• All endpoints must require requireSuperAdmin middleware.
• All actions must log to super_admin_actions table.
• No company data should be returned without explicit admin authorization.
• Pagination required for all large queries.
