Replit ticket list — Performance, reliability, and onboarding

Objective

Implement two focused workstreams:
	1.	Performance and reliability hardening
	2.	Improved first-run onboarding and activation

The goal is to make the platform stable under realistic SME usage and make new users reach value faster.

⸻

Workstream 1 — Performance and reliability hardening

P001 — Build test data generator for load simulation

Priority: Critical
Blocked by: None

Task
Create a repeatable test data generator that can seed realistic platform data for performance testing.

Generate at minimum:
	•	50 companies
	•	10–20 users per company
	•	12 reporting periods per company
	•	all default metrics populated
	•	raw data inputs across periods
	•	evidence files across periods
	•	action plans
	•	policies and policy versions
	•	reports
	•	questionnaires and answers
	•	notifications
	•	background jobs
	•	audit logs
	•	activity logs

Files likely impacted
	•	server scripts or seed utilities
	•	shared/schema.ts if any seed-related support is needed

Acceptance criteria
	•	A script can generate realistic multi-company test data in one command
	•	Seeded data includes historical periods and linked records
	•	The script can be run safely in non-production environments only
	•	Output includes counts of records created per major table

P005 — Harden background job scheduler

Priority: Critical
Blocked by: P002

Task
Review and harden the scheduler and queued job execution.

Ensure:
	•	jobs do not run twice accidentally
	•	retries are safe
	•	stuck jobs can be detected
	•	long-running jobs are visible
	•	failed jobs are logged clearly
	•	imports and report generation do not block normal request handling

Add or verify support for:
	•	payload
	•	attempts
	•	maxAttempts
	•	lockedAt
	•	workerId
	•	idempotencyKey

Acceptance criteria
	•	Duplicate execution is prevented for the same queued job
	•	Failed jobs retry within defined limits
	•	Stuck jobs can be identified from the admin/system view
	•	Report generation and import processing work through the job layer where appropriate
	•	Job runs and failures are visible in logs or health UI

Workstream 2 — First-run onboarding and activation

O001 — Redesign onboarding flow structure

Priority: Critical
Blocked by: None

Task
Create a new guided first-run onboarding flow for new companies.

Flow should include these steps:
	1.	Complete company profile
	2.	Choose ESG focus areas
	3.	Set up reporting basics
	4.	Enter first data
	5.	Upload first evidence
	6.	Generate first output

Design it as a guided success journey, not only a form wizard.

Acceptance criteria
	•	New users are routed into onboarding after signup/first login
	•	Onboarding can be resumed later
	•	Progress is stored per company
	•	Users can clearly see current step, completed steps, and next step
