Please implement the next stage of the ESG Manager product improvements.
These changes should be implemented carefully in sequence and must not break the existing application.

The platform is already live and running on a production server, so stability and backward compatibility are critical.

⸻

PROJECT GOAL

Improve usability for SME users by adding:
	1.	A richer guided onboarding experience
	2.	Clear contextual guidance on key pages
	3.	A top-level ESG programme progress dashboard
	4.	An in-app contextual support assistant

These features should make it extremely easy for a non-expert SME to understand:

• what ESG means for them
• what they need to do next
• how far along their programme is
• how to use each section of the platform

⸻

IMPLEMENTATION ORDER

Please implement the following in this exact order:
	1.	Guided Onboarding Wizard Improvements
	2.	Contextual Page Guidance Panels
	3.	ESG Programme Status Dashboard
	4.	In-App Support Assistant

Do NOT attempt to implement all logic in one pass.
Each stage should integrate cleanly with the previous one.

⸻

	1.	GUIDED ONBOARDING WIZARD IMPROVEMENTS

Enhance the onboarding experience so the platform generates a personalised ESG action plan.

Key changes:

• Replace the existing maturity dropdown with a short 5-question maturity assessment
• Questions should calculate a maturity level:
	•	Starter
	•	Developing
	•	Established

• Update the topic selection step so each ESG topic includes a short benefit statement explaining the business value

Example:
Climate Change — “Reduce energy costs and meet customer carbon requirements”

• Add a final onboarding step called:

“Your ESG Action Plan”

This step should show:

Top 3 recommended policies
Top 5 recommended metrics
Top 3 evidence items to collect
Suggested reporting frequency

These recommendations should be rule-based and derived from:

maturity level + selected ESG topics

NO AI calls should be used here.
