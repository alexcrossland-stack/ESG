Please proceed with Plan #29 (Platform Super Admin Console) but update the plan with the following improvements before implementation.

1. Role enum migration
Do not rely primarily on startup SQL to add the super_admin role. Instead update the role enum in shared/schema.ts and apply it through the normal schema migration process. Only fall back to startup SQL (ALTER TYPE role ADD VALUE IF NOT EXISTS 'super_admin') if the migration tool cannot safely update the enum.

2. Add impersonation exit endpoint
Add a new endpoint:
POST /api/admin/impersonation/exit

This endpoint must:
- restore the original super admin session
- clear impersonation state
- log the exit action in super_admin_actions

3. Define impersonation session fields
When impersonation is active, store explicit fields in the session:

isImpersonating: true
originalSuperAdminUserId
originalSuperAdminRole
impersonatedCompanyId
(optional) impersonatedUserId

These should be cleared when the exit endpoint is called.

4. Expand audit logging
In addition to suspend/reactivate actions, also log:

view_company
impersonate_start
impersonate_exit

The super_admin_actions table should include:
id
adminUserId
action
targetCompanyId
targetUserId
metadata jsonb
createdAt
(optional) ipAddress
(optional) userAgent

5. Clarify impersonation behaviour
Impersonation should be company-level, not arbitrary user-level.

When a super admin impersonates a company:
- identify the company admin account automatically
- if multiple admins exist choose the oldest admin
- if no admin exists return a clear error

6. Add pagination and search
The following endpoints must support pagination and search:

GET /api/admin/companies
GET /api/admin/users

Pagination should default to 50 rows per page.
Search should support company name and user email.

7. Define company detail view
The endpoint GET /api/admin/company/:companyId should power either:

/admin/companies/:companyId (preferred)
or a modal/slide-over detail view

The view should show:
company metadata
plan tier
users
policy/metric/report counts
recent activity

8. Suspension enforcement
When companies.status = "suspended":

- login must be blocked
- authenticated API access must also be blocked
- existing sessions should be prevented from continuing to use the platform

Enforce this check in authenticated middleware.

9. Add safety guardrails
Prevent the following situations:

- impersonating another super admin
- demoting or removing the last super admin
- suspending the last super admin context if applicable

10. Admin UI improvements
The admin dashboard should include:

Overview cards:
- total companies
- total users
- pro subscriptions
- platform errors last 24h

Companies table:
- search
- pagination
- actions (View, Impersonate, Suspend, Reactivate)

Users table:
- search
- pagination
- company reference
- role
- status

When impersonation is active show a sticky banner:

"Viewing as [Company Name] (Impersonation Mode)"
with a "Return to Admin" button that calls /api/admin/impersonation/exit.

All admin routes must be protected by requireSuperAdmin middleware and frontend guards must redirect non-super-admin users away from /admin.
