I want you to perform a thorough end-to-end testing, debugging, and real-user journey validation pass of this platform.

Primary objective:
Test the platform both technically and from the perspective of a real first-time user setting up and using the platform from scratch.

Your responsibilities:

1. Understand the full system
- Review the full codebase, architecture, frontend routes, backend endpoints, database schema, auth model, background jobs, permissions, integrations, and environment dependencies.
- Identify the main user types and their expected journeys through the platform.

2. Create a complete test plan
Cover:
- unit tests
- integration tests
- end-to-end tests
- manual exploratory testing
- first-time user onboarding testing
- role/permission testing
- validation and error handling
- mobile/responsive behavior
- accessibility basics
- performance bottlenecks
- logging and observability gaps

3. Test the platform as a real user from scratch
I specifically want you to simulate a genuine user setting up and using the platform, including:
- landing on the product with no prior knowledge
- signing up
- verifying email if applicable
- logging in
- going through onboarding
- understanding what to do next from the UI alone
- setting up the company/account
- entering initial business data
- configuring settings
- inviting team members if applicable
- connecting any integrations if applicable
- using the core product workflow from beginning to end
- returning later and continuing work
- upgrading or encountering plan limits if applicable
- exporting, saving, submitting, or completing the main outcomes the platform is designed for

4. Evaluate usability as well as technical correctness
During user testing, assess:
- whether the UI makes the next step obvious
- whether instructions are clear
- whether terminology is understandable for an SME/non-technical user
- whether important actions are easy to find
- whether there are confusing dead ends
- whether the user could reasonably complete setup without support
- where friction, hesitation, ambiguity, or cognitive overload occurs
- whether empty states, success states, and error states are helpful

5. Actively try to break the platform
Do adversarial and edge-case testing including:
- invalid inputs
- missing required data
- duplicate submissions
- stale UI state
- page refreshes mid-flow
- browser back/forward behavior
- expired sessions
- unauthorized access attempts
- incorrect role access
- malformed payloads
- large uploads
- slow network simulation
- API timeouts
- concurrent edits
- pagination boundaries
- date/time/timezone edge cases
- mobile viewport issues

6. For every issue found
- reproduce it
- explain expected vs actual behavior
- identify root cause
- implement a fix
- verify the fix
- add or improve regression coverage
- document what changed

7. Prioritise ruthlessly
Classify all issues as:
- Critical = blocks signup, onboarding, core workflow, billing, security, or data integrity
- High = major friction or broken functionality in important flows
- Medium = confusing UX, partial bugs, inconsistent behavior
- Low = cosmetic or minor polish issues

Fix critical and high issues first.

8. Do not stop at the first pass
- Continue testing after each round of fixes
- Re-run the affected flows
- Re-test for regressions
- Keep iterating until core journeys pass cleanly

Core journeys to test at minimum:
- signup / login / logout
- password reset
- first-time onboarding
- dashboard first load
- initial account/company setup
- core product data entry and editing
- search / filtering / sorting
- reports / exports
- file upload / download
- email / notification flows
- role-based access control
- admin flows
- billing / upgrade prompts if applicable
- error recovery and retry handling
- returning user workflow

Requirements:
- Do not rely only on static code review.
- Actually run and exercise the application.
- Test both happy path and failure path.
- Test from the viewpoint of a non-technical business user.
- Assume there are hidden bugs and UX problems.
- Be skeptical and try to break the experience.

Deliverables:
1. Test plan
2. Bug report table with:
   - ID
   - severity
   - area
   - user journey step
   - reproduction steps
   - expected result
   - actual result
   - root cause
   - fix implemented
   - regression test added
   - status
3. UX friction report with:
   - confusing screens
   - unclear wording
   - missing guidance
   - dead ends
   - unnecessary complexity
   - recommendations to improve completion rate
4. List of commands run
5. Summary of code changes
6. Remaining risks / known issues
7. Final QA sign-off summary

Important:
I want this to include “can a normal user successfully set up and use this platform without help?” not just “do the features technically work?”
