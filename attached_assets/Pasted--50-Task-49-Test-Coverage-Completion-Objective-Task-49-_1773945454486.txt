#50 - Task #49 Test Coverage Completion

## Objective

Task #49 ("First-time activation audit & fix") is merged, but its current test suite only covers unauthenticated baselines.

The goal of this task is to close the remaining validation gap by implementing the full regression suite required by Task #49:
- authenticated cross-tenant isolation
- RBAC enforcement
- session lifecycle enforcement
- malformed-payload validation expansion
- Playwright end-to-end coverage for the key first-time activation journeys
- shared fixture/seed utilities and package scripts so the full suite can be run with one command

This task is about proving that the fixes from #49 actually hold under authenticated, role-based, and end-to-end conditions.

## Required operating approach

Work in this order.

### Phase 1 — Review existing coverage and route surface
Before writing new tests:
- Review the current `tests/api-security.test.ts`
- Review relevant protected and write-capable routes in `server/routes.ts`
- Identify which routes are already covered and which remain untested
- Map each required assertion below to at least one concrete route

Do not guess coverage. Confirm it from the code.

### Phase 2 — Build deterministic seed/fixture support
Create a shared fixture/seed utility that can be reused by both API tests and Playwright setup.

Requirements:
- deterministic tenant/account creation
- no dependence on production data
- no manual setup required
- reusable auth/session state
- safe teardown or isolation by unique test prefix

### Phase 3 — Add API regression suites
Extend the current API suite with authenticated tenant isolation, RBAC, session lifecycle, and malformed-payload coverage.

### Phase 4 — Add Playwright E2E coverage
Add browser-level tests for the first-time activation flows listed below.

### Phase 5 — Standardize execution
Add `npm` scripts so the full suite can be run locally in a consistent way:
- `test:api`
- `test:e2e`
- `test:all`

### Phase 6 — Document coverage
Add a short summary file documenting what is now covered and what remains out of scope.

## Done looks like

### API coverage
- Authenticated cross-tenant isolation tests pass:
  - a user from Company A cannot read or write Company B resources even with a valid session and known resource IDs
  - coverage includes metrics, data-entry, topics, reports, policies, questionnaires, and related actions where routes exist
- RBAC tests pass:
  - viewer-role users receive `403` on all write-protected endpoints tested
  - contributor-role users are rejected from admin/settings/template-admin routes where applicable
  - admin-role callers are rejected from super-admin-only routes
- Session lifecycle tests pass:
  - a cookie from a logged-out session is rejected with `401` on every protected route attempted afterwards
  - fabricated or malformed session tokens return `401`, never `500`
- Malformed-payload tests pass:
  - write endpoints return `400` with a safe validation error for missing required fields, wrong types, invalid values, and oversized payloads
  - no malformed request tested should produce `500`

### Playwright coverage
The following Playwright specs pass against the running local dev server:
- signup → login → logout
- onboarding minimum path and Quick Start dismissal
- first metric entry using the raw-data input flow
- dashboard first-run empty states and CTA navigation
- first report generation with preview or list appearance
- viewer-role restrictions in both UI and direct in-browser API call attempts

### Shared utilities and scripts
- A shared fixture/seed utility provisions two isolated tenants deterministically:
  - Tenant A: admin + viewer user
  - Tenant B: admin user + known resource IDs
- The fixture utility can be used by both API tests and Playwright global setup
- `package.json` includes:
  - `test:api`
  - `test:e2e`
  - `test:all`
- `tests/COVERAGE.md` documents what is covered and what remains uncovered
