Build three AI-assisted features inside my SME ESG web app:
	1.	AI-Assisted Automated ESG Policy Generator for SMEs
	2.	Supplier Questionnaire Autofill

FEATURE 1: AI-ASSISTED AUTOMATED ESG POLICY GENERATOR FOR SMES

Build a guided ESG Policy Generator that helps an SME create a professional ESG policy from a questionnaire.

Goal

Allow a user to answer a short set of business questions and automatically generate a tailored ESG policy draft that they can edit, save, version, approve, and export as PDF.

User flow
	1.	User clicks “Create ESG Policy”
	2.	User completes a guided form
	3.	App generates a draft ESG policy
	4.	User edits the draft in a rich text editor
	5.	User saves a version
	6.	User approves and publishes the policy
	7.	User exports PDF

Questions the generator should ask

Company profile:
	•	company name
	•	sector
	•	country
	•	number of employees
	•	number of sites
	•	service business, distributor, or manufacturer
	•	whether the business has vehicles
	•	whether the business has suppliers outside its home country

Environmental questions:
	•	do you track electricity use
	•	do you track fuel or gas use
	•	do you track water use
	•	do you track waste and recycling
	•	do you want to include a carbon reduction commitment
	•	do you have any environmental certifications

Social questions:
	•	do you have an employee handbook
	•	do you track diversity data
	•	do you provide employee training
	•	do you track health and safety incidents
	•	do you have wellbeing initiatives
	•	do you pay at least living wage or equivalent

Governance questions:
	•	do you have anti-bribery rules
	•	do you have a whistleblowing process
	•	do you have a data privacy policy
	•	who is responsible for ESG internally
	•	how often should the policy be reviewed

Output

Generate an ESG policy with these sections:
	•	title page
	•	purpose and scope
	•	ESG commitment statement
	•	environmental commitments
	•	social commitments
	•	governance commitments
	•	responsibilities
	•	implementation and monitoring
	•	review cycle
	•	approval section

Content rules
	•	Write in plain English
	•	Make it sound professional but not overly legal
	•	Use the company’s profile and answers to tailor the content
	•	Where data is missing, use sensible default wording
	•	Include optional placeholders such as “[Insert review month]” only where necessary
	•	Do not give legal advice
	•	Add a footer note that the policy should be reviewed by management before formal adoption

Functional requirements
	•	editable generated text
	•	save draft
	•	save version history
	•	mark one version as current
	•	PDF export
	•	show last updated date
	•	show next review date
	•	audit log for creation and edits

Technical implementation

Create:
	•	policy generation form
	•	backend endpoint to generate policy text from form inputs
	•	policy editor page
	•	policy version table
	•	PDF export endpoint

Create database tables:
	•	esg_policies
	•	policy_versions
	•	policy_generation_inputs
	•	audit_logs

FEATURE 2: SUPPLIER QUESTIONNAIRE AUTOFILL

Build a feature that helps SMEs answer customer, lender, procurement, and tender ESG questionnaires using data already stored in the app.

Goal

Allow users to paste questions, upload a questionnaire, or enter questions manually, and then generate suggested answers based on the company’s ESG policy, metrics, actions, and uploaded evidence.

User flow
	1.	User clicks “Supplier Questionnaire Autofill”
	2.	User either:
	•	pastes a list of questions
	•	uploads a document
	•	enters questions manually into a form
	3.	App extracts the questions
	4.	App matches each question against available company ESG data
	5.	App generates a suggested answer for each question
	6.	User edits answers
	7.	User exports completed responses

Supported questionnaire examples
	•	customer supplier ESG questionnaires
	•	procurement forms
	•	tender ESG response sections
	•	lender ESG due diligence requests
	•	investor information requests

Input handling

Support:
	•	plain text paste
	•	CSV upload
	•	Word document upload
	•	PDF upload if text can be extracted
	•	manual question-by-question entry

For each question, the system should:
	•	classify the question into Environmental, Social, Governance, Carbon, Policy, Training, Supply Chain, Data Privacy, etc.
	•	search the company’s stored data
	•	suggest an answer
	•	show confidence level: high, medium, low
	•	show the source of the answer, for example:
	•	ESG policy
	•	metric value
	•	action plan
	•	evidence file
	•	no source found

Answer generation rules
	•	write concise, professional answers
	•	avoid inventing facts
	•	if data is missing, say so clearly and suggest a safe placeholder answer
	•	answers must be editable
	•	keep tone suitable for supplier/customer submissions
	•	where relevant, include current metric values
	•	where relevant, refer to policies and commitments already stored in the app

Example logic

If the question is:
“Do you have an ESG policy?”
The system should check if an ESG policy exists and answer accordingly.

If the question is:
“Do you measure carbon emissions?”
The system should check whether carbon calculator data exists.

If the question is:
“Describe your diversity and inclusion approach.”
The system should use policy text and social metrics if available.

If the question is:
“Do you monitor supplier standards?”
The system should use governance policy content or supplier code content if available.

Output screen

For each question show:
	•	original question
	•	category
	•	suggested answer
	•	confidence level
	•	source references
	•	editable text box
	•	approve / reject / rewrite buttons

Export options
	•	CSV
	•	Word
	•	copy-to-clipboard formatted output

Technical implementation

Create:
	•	questionnaire upload page
	•	question parser
	•	question categoriser
	•	answer suggestion engine
	•	editable answer review screen
	•	export functions

Create database tables:
	•	questionnaires
	•	questionnaire_questions
	•	questionnaire_answers
	•	answer_sources
	•	uploads

Matching logic

Build a rules-based matching engine first. Do not depend entirely on AI.
Use deterministic logic such as:
	•	if question contains “policy”, search policy documents
	•	if question contains “carbon”, search calculator results
	•	if question contains “training”, search social metrics and governance training data
	•	if question contains “waste”, search environmental metrics
	•	if question contains “anti-bribery”, search governance commitments

Then optionally enhance with AI-generated answer drafting.

Important rule

Never make up company claims. If no evidence exists, say:
“No formal data is currently recorded in the system for this area. A draft response can be prepared for management review.”

FEATURE 3: SIMPLE CARBON CALCULATOR

Build a simple SME-friendly carbon calculator that estimates basic Scope 1 and Scope 2 emissions, plus selected business travel if possible.

Goal

Allow a user to enter simple operational data and get a basic emissions estimate suitable for SME reporting.

User flow
	1.	User clicks “Carbon Calculator”
	2.	User enters activity data
	3.	App calculates emissions
	4.	App shows totals and breakdown
	5.	App stores the result by reporting period
	6.	App lets the user include results in reports and questionnaires

Input fields

Energy and fuel:
	•	electricity consumption
	•	electricity unit: kWh
	•	gas consumption
	•	gas unit: kWh or litres if relevant
	•	diesel consumption
	•	petrol consumption
	•	company vehicle mileage
	•	business travel by air
	•	business travel by rail
	•	business travel by hotel nights if you want optional travel estimates

Business profile fields:
	•	reporting period
	•	country
	•	number of employees
	•	revenue optional
	•	floor area optional

Output

Show:
	•	Scope 1 emissions
	•	Scope 2 emissions
	•	optional travel emissions
	•	total emissions
	•	emissions per employee
	•	trend versus previous period if data exists

Functional rules
	•	keep it simple
	•	explain each input in plain English
	•	allow monthly, quarterly, or annual entry
	•	store raw inputs and calculated outputs
	•	allow manual override only by admin
	•	allow export to reports

Calculation logic

Create a calculation engine with emissions factors stored in a table so they can be updated later.

For MVP:
	•	electricity factor
	•	natural gas factor
	•	diesel factor
	•	petrol factor
	•	flight factor
	•	rail factor

Create a simple emissions factor table with:
	•	factor name
	•	country
	•	unit
	•	factor value
	•	effective date
	•	source label

Important implementation note

Build the calculator so the factors are configurable in the database, not hardcoded directly into the UI.

Results page

Show:
	•	emissions summary card
	•	breakdown by source
	•	reporting-period comparison
	•	explanation of assumptions
	•	warning that calculations are estimates and should be reviewed for formal disclosure use

Technical implementation

Create:
	•	carbon calculator form
	•	calculation service
	•	emissions factor table
	•	result history page
	•	chart of emissions over time

Create database tables:
	•	carbon_activity_inputs
	•	carbon_results
	•	emission_factors
	•	reporting_periods
